# Правила для AI Агентов

Эти правила обязательны к исполнению для любого AI агента, работающего над проектом **Nano Banana Bot**, для обеспечения согласованности, качества и поддерживаемости кода.

## 1. Контекстная осведомленность
- **Всегда** читайте `README.md` и `ROADMAP.md` перед началом крупной задачи.
- Проверяйте `CHANGELOG.md`, чтобы понимать историю последних изменений.

## 2. Стандарты кода
- **Чистый код**: Пишите самодокументируемый код с понятными именами переменных и функций.
- **Комментарии**: Комментируйте сложную логику, но избегайте излишних комментариев для очевидного кода. **Все комментарии должны быть на русском языке.**
- **Типизация**: Используйте подсказки типов (type hinting для Python) или TypeScript (если применимо).
- **Обработка ошибок**: Реализуйте надежную обработку ошибок, особенно для вызовов API (Nano Banana, Telegram).

## 3. Документация
- **Обновление Changelog**: Записи в `CHANGELOG.md` должны добавляться после завершения фичи или исправления.
- **Docstrings**: Все публичные функции и классы должны иметь docstrings на русском языке.

## 4. Взаимодействие с пользователем
- **Уточняйте**: Если требование неоднозначно (например, "Сделай красиво"), спросите пользователя о предпочтениях или примерах.
- **Предлагайте планы**: Перед написанием большого объема кода предложите план или архитектуру в `implementation_plan.md`.

## 5. Управление файлами
- **Избегайте мусора**: Не создавайте временные файлы в корневой директории, если это не необходимо. Используйте `tmp/` или папки сборки.
- **Атомарные правки**: Старайтесь фокусировать изменения файлов на текущей задаче.
